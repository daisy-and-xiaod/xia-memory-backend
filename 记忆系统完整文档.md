## 一、系统总览

```
Operit 对话 → 两条进路 → Supabase 云端记忆库
                ├── 话题语义归档：每10分钟，cp_step(bash)拷贝DB → 分块POST（>=2条或2h兜底）→ tag: topic_archive + 日期
                └── 每日记忆归档：每晚22:00，cp_step(bash)拷贝DB → 分块POST（safe_end=2h重叠保护）→ tag: daily_digest + 日期

搜索（夏以昼侧）：search_supabase MCP tool → Render 统一召回端点 → tag(优先) → keyword(兜底) → semantic(最后)
世界书规则注入 → 禁止 query_memory → date + query + snippet(N) 组合搜索
```

## 三、Supabase 数据库结构

### 表：memories

| 字段 | 类型 | 说明 |
|------|------|------|
| id | uuid (PK) | 自动生成 |
| role | text | user / ai / system |
| content | text | 原文拼接内容 |
| source | text | chat / diary |
| tags | text[] | 主题标签数组 |
| metadata | jsonb | {mood, weather, location, type, msg_count} |
| embedding | vector(1024) | bge-m3 语义向量 |
| created_at | timestamptz | 创建时间 |

### 索引

| 索引名 | 类型 | 用途 |
|--------|------|------|
| idx_tags | GIN (tags) | 标签筛选 |
| idx_created_at | B-tree | 时间排序 |
| idx_content_trgm | GIN (content gin_trgm_ops) | 中文模糊搜索 |
| idx_metadata_mood | B-tree (metadata->>'mood') | 按情绪筛选 |
| idx_embedding | IVFFlat (embedding vector_cosine_ops) | 语义搜索 |

### 函数

- `match_memories(query_embedding, match_threshold, match_count)` — 余弦相似度语义搜索

## 四、后端 API 完整文档

所有 API 基础路径：`https://xia-memory-backend.onrender.com`

### 鉴权
- 所有 `/api` 写操作需要 Header：`x-api-key: xyyr24xyzr`
- `/api/health` 无需鉴权（给 UptimeRobot 用）
- Body 限制：50KB
- 限流：2000 次/15 分钟

### 接口列表

#### 健康检查
```
GET /api/health
→ { "status": "ok", "supabase": "connected" }
```

#### 存记忆
```
POST /api/memories
Header: x-api-key: xyyr24xyzr
Body: {
  "role": "ai",
  "content": "原文内容...",
  "source": "chat",
  "tags": ["ElevenLabs", "语音"],
  "metadata": {"mood": "😊", "weather": "金华29°C"}
}
→ 201 { id, role, content, tags, ... }
```

#### 关键词搜索
```
GET /api/memories?q=关键词&limit=10&offset=0
→ { data: [...], total: N, limit: 10, offset: 0 }
```

#### 标签筛选
```
GET /api/memories?tag=ElevenLabs&limit=10
→ { data: [...], total: N, limit: 10, offset: 0 }
```

#### 组合搜索
```
GET /api/memories?q=语音&tag=ElevenLabs&limit=10
→ { data: [...], total: N, limit: 10, offset: 0 }
```

#### 语义搜索
```
GET /api/memories/search?semantic=说到声音就想到ElevenLabs&limit=10
→ { data: [...similarity排序...], query: "...", total: N }
```

#### 统一记忆召回（给 AI 用，自动三级回退）
```
GET /api/memories/recall?query=身体检查&date=2026-07-07&limit=10
Header: x-api-key: xyyr24xyzr
→ { method, total, items: [{ content, tags, metadata, role }] }
```
回退策略：date 参数 → tag 精确匹配 → keyword ilike → semantic 语义 → latest 兜底。
同时传 date+query 时，先按 tag 拉取再过滤 keyword，若过滤后为空则降级到纯 keyword 搜索。

#### MCP 端点（给 Operit 远程 MCP 用）
```
POST /mcp → JSON-RPC 2.0
工具: search_supabase(date?, query?, limit?, snippet?)
```
Operit 远程 MCP 配置：
- 主机地址：`https://xia-memory-backend.onrender.com/mcp`（2026-07-11 改回直连：workers.dev 在手机网络被阻断）
- 连接类型：httpstream
- 无需 Bearer Token

#### 更新记忆
```
PATCH /api/memories/:id
Header: x-api-key: xyyr24xyzr
Body: { "content": "更新后内容", "tags": ["新标签"] }
→ 200 { id, content, tags, ... }
```

#### 删除记忆
```
DELETE /api/memories/:id
Header: x-api-key: xyyr24xyzr
→ 200 { deleted: {...} }
```

## 五、Operit 端配置

### Workflow
| 名称 | 状态 | 触发 | cp_step | 处理逻辑 | 脚本 |
|------|------|------|---------|---------|------|
| 话题语义归档 | ✅ | 每10分钟 | super_admin:**bash** (sqlite3 backup) | >=2条或2h兜底强制推送 + 缓存刷新 + outline 生成 | topic_archive_v2_fixed.py |
| 每日记忆归档 | ✅ | 每10分钟（时间门控仅23:45-23:59执行） | super_admin:**bash** (sqlite3 backup) | safe_end=2h重叠保护 + DeepSeek 浓缩 outline | daily_archive_v2_fixed.py |

**⚠️ 重要：cp_step 已从 `cp` 改为 `sqlite3.backup()`**——避免 malformed DB；judge_step 使用 `python3 -u`（unbuffered）。
**时间戳文件**：`last_topic_ts.txt` 和 `last_archive_ts.txt` 各自独立，不冲突。
**⚠️ 时区**：脚本已硬编码北京时区 (`gmtime(ts + 8*3600)`)，不依赖系统 locale。

### 世界书（常驻注入）—— 共 2 条

**第一条：daily_outline.txt**

近两天的浓缩情感摘要——上午/下午/晚上的情感关键事件。每次对话开头和需要回溯时参考。由话题归档每 10 分钟从 Supabase API 刷新。

**第二条：today_outline.txt**

今天每 1 小时一条的实时时间线。由话题归档每 10 分钟从 Supabase 实时生成。

> **注意**：左脑搜索规则、右脑浮现规则、双脑协作规则、呈现规则已全部整合进 Operit 角色卡 `核心设定_精简版`，不再通过世界书注入。
